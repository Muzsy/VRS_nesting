# SGH-Q44 ‚Äî Per-attempt multisheet timing and CDE diagnostics for Q42

**Task kind:** diagnostics / instrumentation / measurement only.
No solver-strategy, CDE-optimisation, sheet-count-objective or warm-start change was made.

## 0. Summary of what was added

The `sparrow_cde_multisheet` pipeline previously surfaced only **aggregated** metrics, so it
was impossible to prove which subset-attempt (1-sheet / 2-sheet / 3-sheet full pool) consumed
the runtime and the CDE work in the Q42 runs. SGH-Q44 adds a **per-attempt diagnostics record**
emitted for every `run_core_attempt` invocation inside
[`run_finite_stock_multisheet`](../../../rust/vrs_solver/src/optimizer/sparrow/multisheet.rs).

Each attempt is bracketed by a thread-local CDE-counter snapshot (before/after), a wall-clock
timer, and a capture of that attempt's `SparrowDiagnostics`. The records are serialised in the
solver output under:

```json
{ "optimizer_diagnostics": { "sparrow_ms_attempt_diagnostics": [ ... ] } }
```

with the stable aggregate companion fields:

```text
sparrow_ms_attempts
sparrow_ms_candidate_subsets
sparrow_ms_attempt_diagnostics_count            (= 3 for Q42)
sparrow_ms_attempt_diagnostics_schema_version   (= 1)
```

The instrumentation is purely additive: the single-threaded solve path means the thread-local
CDE counters that already feed the aggregated `collision_backend_diagnostics` are snapshotted at
the attempt boundaries, so the **per-attempt deltas sum to the aggregate counters** (proven by a
unit test and re-confirmed on the production run ‚Äî see ¬ß1 / ¬ß4).

### `stop_reason` vocabulary (introduced by Q44)

Each attempt gets an explicit reason ‚Äî never just "not feasible":

- `valid_full_solution` ‚Äî feasible, every instance placed (pairs=0, boundary=0).
- `partial_sanitized` ‚Äî core ran out its budget without resolving all collisions; colliding
  instances removed by the greedy-MIS sanitize to emit a collision-free partial.
- `partial_pre_unplaced` ‚Äî core feasible-shaped but some instances were pre-unplaced
  (never-fits-stock / unsupported geometry); no collisions.
- `core_infeasible` ‚Äî core problem could not be built / invoked.
- `other` ‚Äî none of the above.

Budget pressure is reported quantitatively per attempt via `allocated_time_limit_s`,
`actual_runtime_ms`, `remaining_budget_after_s` and `deadline_hit_after_attempt`, so a
`partial_sanitized` attempt that exhausted its budget is identifiable as a timeout without
overloading the `stop_reason` token.

## Methodology

- **Attribution boundary.** `cde_before` is snapshotted immediately before `run_core_attempt`;
  `cde_after` immediately after this attempt's incumbent update (so it also captures the
  sanitize-path CDE work for that attempt). Between one attempt's `cde_after` and the next
  attempt's `cde_before` no CDE work happens (only subset selection / time-budget arithmetic),
  so consecutive deltas tile the whole solve with no gaps or overlaps.
- **Single-thread validity.** The Sparrow solve and its worker competition run on the calling
  thread (no `thread::spawn` / rayon in the sparrow / CDE path), so the thread-local
  `cde_observability` counters ‚Äî the same ones that feed the aggregated
  `collision_backend_diagnostics` ‚Äî are the authoritative per-attempt accounting.
- **Consistency contract.** Œ£(per-attempt CDE delta) == aggregate `collision_backend_diagnostics`
  counter, minus a small post-loop residual (the margin/spacing final validators, which do not
  touch the CDE batch counters). The Q44 extractor prints the residual explicitly (¬ß1).

## 7. Stop-reason analysis

Per-attempt `stop_reason` (introduced by Q44 ‚Äî never just "not feasible"):

| run | attempt | stop_reason | meaning |
| --- | ---: | --- | --- |
| 1200/2400 | 0 (1-sheet) | `partial_sanitized` | core hit collisions it could not resolve in budget (445 pairs); colliding items removed by greedy-MIS sanitize ‚Üí collision-free 145-part partial. |

The 2-sheet attempts are budget-bounded timeouts (allocated 600/1200 s, consumed 615/1216 s,

after the probe + before the full pool). `allocated_time_limit_s` vs `actual_runtime_ms` plus

*outcome* (partial-sanitized) rather than collapsing it to "not feasible".

## 8. Final conclusion

- **Main computational cost in Q42 is the 2-sheet subset attempt.** It consumes 88‚Äì93 % of
  multisheet runtime and 72‚Äì98 % of every CDE counter, yet it only ever produces a *sanitized
  partial* (238‚Äì240 / 276 on two sheets) that is discarded once the 3-sheet attempt finds the full
  solution.
- **The 2-sheet attempt behaviour is a budget-bounded timeout.** It is allocated 50 % of the total
  budget (600 s at 1200 s; 1200 s at 2400 s), runs essentially all of it (615 s / 1216 s), never
  reaches feasibility (always leaves ~96‚Äì98 residual collision pairs), and is then sanitized. Its
  GLS iteration count scales linearly with the budget (383 ‚Üí 756), so its cost scales with the
  budget too.
- **The full-pool 3-sheet behaviour is fast and deterministic.** It reaches the feasible 276-part
  layout in ~34 s and converges (28 iterations), byte-identical across both runs and the baseline.
  The large remaining-budget allocation for the full pool (442 s / 1042 s) is never used.
- **The previously aggregated diagnostics were misleading because** every `sparrow_*` /
  `cde_*` counter in the output is sourced from the *best-core* (full-pool) attempt or summed over
  the whole run, so the headline numbers blended a cheap 34 s deterministic feasible attempt with
  a 615‚Äì1216 s partial attempt. There was no way to see that ~90 % of the compute went into an
  attempt whose result is thrown away, nor that the 1200‚Üí2400 delta lands entirely in that one
  attempt.
- **Recommended next development direction based on evidence** (statement only, not a new task):
  the highest-leverage saving is the 2-sheet attempt's time-budget ‚Äî it is the dominant cost and
  never contributes the incumbent on this fixture; the full-pool attempt already solves Q42 in
  ~34 s. This is an observation about where the cost sits, not an instruction to change the
  solver, the budget policy, or the sheet-count objective.
